# 変更履歴

## Ver2.2

- アカウント設定に、[PCA Arch ポータルへの導線（リンク）](/docs/apps/account/account-common.md#%E3%83%A6%E3%83%BC%E3%82%B6%E3%83%BC%E6%83%85%E5%A0%B1)を追加しました
- PCA Arch 契約向けのシステム連携に対応しました
  - [PCA Arch サービスの体験利用](https://pca.jp/ap/pcaarchtrial)に申し込みがあると、社内システムを介して、PCA Arch のサービス利用をセットアップします
- システムのモニタリング環境を整備しました
  - AWSの主要サービスやID管理サーバーのログやメトリクスを分析できるようにしました
  - 定義された値を超えるシステムの実行状態を検出すると開発・運用チームに通知が届くようにしました

## Ver2.1

- PCA Arch 向けの基盤を整備しました
  - PCA Arch ポータルの認可機能に対応しました
  - PCA Arch 契約向けのシステム連携の準備を進めました
- ログイン画面に「[＜ログインできない場合はこちら＞](/docs/common/ユーザー認証.md#1-%E3%83%A6%E3%83%BC%E3%82%B6%E3%83%BC%E5%85%A5%E5%8A%9B)」リンクを設置しました
- Ver2.0.5 では不安定だった機能を改善しました
  - [初回の組織管理者アカウントの回復手順](/docs/manual/ご利用までの流れ/初回の組織管理者アカウントを設定する.md#%E3%82%A2%E3%82%AB%E3%82%A6%E3%83%B3%E3%83%88%E8%A8%AD%E5%AE%9A%E3%81%AE%E5%9B%9E%E5%BE%A9%E6%89%8B%E9%A0%86)を改善しました
  - [ユーザー情報の同期](/docs/manual/組織管理者向け/ユーザーを管理する/ユーザーのアカウントを管理する.md#7-%E3%83%A6%E3%83%BC%E3%82%B6%E3%83%BC%E6%83%85%E5%A0%B1%E3%82%92%E5%90%8C%E6%9C%9F%E3%81%99%E3%82%8B)の安定性を改善しました

## Ver2.0.5

- 組織管理者向けの保守機能を追加しました
  - [バウンスメールアドレスを確認できる](/docs/manual/組織管理者向け/ユーザーを管理する/バウンスメールアドレスを確認する.md)ようになりました
  - [臨時のバックアップコードを発行できる](/docs/manual/組織管理者向け/ユーザーを管理する/ユーザーのアカウントを管理する.md#3-ユーザーのバックアップコードをリセットする)ようになりました
  - [明示的にユーザー情報を同期できる](/docs/manual/組織管理者向け/ユーザーを管理する/ユーザーのアカウントを管理する.md#7-ユーザー情報を同期する)ようになりました
  - [複数組織への所属状態を確認できる](/docs/manual/組織管理者向け/ユーザーを管理する/ユーザー情報を変更する.md#6-ユーザー情報の変更)ようになりました
- 各種メッセージを改善しました
  - メールアドレス変更のメール本文に、[期限切れの場合の案内](/docs/common/メール送信.md#メールアドレス検証)を追加しました
  - [パスワード要件を画面上に表示する](/docs/manual/一般ユーザー向け/パスワードを変更する.md#5-新しいパスワードの入力)ようにしました
  - [共有アカウントの作成に対する注意メッセージ](/docs/manual/組織管理者向け/ユーザーを管理する/ユーザーのアカウントを作成する.md#4-ユーザー作成の開始)を追加しました
  - [バックアップコードの発行画面のガイドメッセージ](/docs/manual/一般ユーザー向け/メールでアカウント設定リクエストが届いたら.md#4-バックアップコードの発行)を簡潔にし、コピー時に完了メッセージを追加しました
- ログイン操作のUXを改善しました
  - [ログイン操作で分かりやすくロックアウト状態を表示](https://github.com/pca-idp/pcaid/issues/1480#issuecomment-2975502727)して、無駄なログイン操作を繰り返さないようにしました
  - パスワードミスでロックアウトにならないように、[パスワード変更手順を案内する](https://github.com/pca-idp/pcaid/issues/1439#issuecomment-2948130479)ようにしました
  - [ボタンを押したことが確実に分かる動作](https://github.com/pca-idp/pcaid/issues/1464#issuecomment-2969036630)にして、ボタン操作ミスを発生しないようにしました
  - [アカウントが有効になっていない状況のメッセージを改善し、案内を追加](https://github.com/pca-idp/pcaid/issues/1490#issuecomment-2989755713)しました
  - [2段階認証メールに送信時刻](/docs/common/メール送信.md#ログイン認証コードのお知らせ)を記載して、グレイリスティングを判別できるようにしました
  - [2段階認証メールの認証コードが再送信により使えなくなること](/docs/common/メール送信.md#ログイン認証コードのお知らせ)を本文に記載しました
- アカウント設定リクエストの流れを改善しました
  - ユーザーが以前のアカウント設定リクエストを忘れていたら、[もう一度アカウント設定リクエストを送信](/docs/apps/orgs/orgs-user-management.md#ユーザー管理ユーザー作成)します
  - アカウント設定リクエスト中は、[組織管理者がメールアドレスを変更できない](/docs/apps/orgs/orgs-user-management.md#ユーザー管理ユーザー詳細の更新)ようにしました
- 「PCA ID について詳しくはこちら」のリンク先を、[特設サイトからWebマニュアルへ変更](https://github.com/pca-idp/pcaid/issues/1445)しました

## Ver2.0

- PCAクラウド/PCAサブスクとの認証連携が可能となりました
  - サービス責任者の追加しました
- PCAクラウド/PCAサブスクから組織を作成できるようになりました
  - PCA Hub の組織と統合することができます
- メール送信サービスを分離し、PCA ID からのメール送信の可用性を向上させました
  - 宛先に届かないメールアドレスへの耐性を強化しました
  - PCAクラウドや PCA Hub とは異なるサービス基盤を使うことでリスク分散となります

## Ver1.0

- PCA Hub の認証基盤を PCA ID に統合しました
  - 既存の PCA Hub ユーザーが PCA ID へ移行できるようにしました
- 組織管理者向けの管理コンソールを作成しました
- 一般ユーザー向けのアカウント設定ページを作成しました
  - 先進技術であるパスキー認証をサポートしました
