# 14. サービス区画

## サービス区画とは

- サービスごとに分類されるユーザー領域や管理情報
  - PCA Hub テナント・・・PCA Hubテナント名で識別
  - PCAクラウド 区画・・・PCAクラウド 連携コードで識別
    - PCAクラウド 契約ID（「サービスユーザーID」とも呼ぶ）を保持する
  - PCAサブスク 契約・・・PCAサブスク 連携コードで識別
    - PCAサブスク 契約ID を保持する
  - PCA Arch 組織・・・PCA ID の組織IDで識別
    - PCA Arch は組織を最小単位として利用する
  - [お客様サポート](https://pca.jp/area_support/)・・・識別する情報はなし
    - すべての組織を対象として顧客IDのみを扱う
- [組織](./組織.md)に対して利用可能なサービス区画を紐付ける
  - １つの組織が複数の PCA Hub テナントや PCAクラウドの区画と紐づくことがあり得る
  - 例外として、[お客様サポート](https://pca.jp/area_support/)は組織への明示的な紐付けなしで、すべての組織を対象とする

## サービス区画の識別値フォーマット

- ベンダー・サービス・契約を、ピリオドで連携したデータとする
  - ベンダー：現時点（2023/08）では `pca` 固定

### 具体例

- PCA Hub
  - サービス名：`hub`
  - 契約：テナント名（サービス利用時に顧客が決定する）
    - 例：pca.hub.tenant1
    - 例：pca.hub.pca-dev
- PCAクラウド
  - サービス名：`cloud`
  - 契約：連携コード（PCA ID と連携する時に PCA ID が発行する）
    - 連携コードは、英小文字と数字を使った「org-xxxx-xxxx」形式のランダム値とする
    - サービス利用時にPCAが発行する契約ID（＝サービスユーザーID）を属性として保持する
    - 例：pca.cloud.org-81sj-u9vo
- PCAサブスク（契約ユーザーID）
  - サービス名：`subsc`
  - 契約：連携コード（PCA ID と連携する時に PCA ID が発行する）
    - 連携コードの仕様は、PCAクラウドと同様
    - サービス利用時にPCAが発行する契約IDを属性として保持する
    - 例：pca.subsc.org-ydtu-uoag
- PCA Arch
  - サービス名：`arch`
  - 契約：組織ID（組織の作成時に決まる）
    - 例：pca.arch.xxxx-xxx-xxx-xxxx
- お客様サポート
  - サービス名：`support`
  - 契約：`customerid` のみ（2025/07 時点）
    - 例：pca.support.customerid
