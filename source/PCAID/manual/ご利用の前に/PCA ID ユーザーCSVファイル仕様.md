# PCA ID ユーザー CSV ファイル仕様

## データ仕様

- データ形式：CSV（カンマ区切り）
- ファイル拡張子：.csv
- 文字エンコーディング：UTF-8 (BOM付き)
- 改行文字：CRLF または LF
- 行フォーマット
  - 1行目：バージョン情報（省略可）
    - 例：Ver1.0
    - 省略された場合は、最新バージョンと解釈します。
  - 2行目：ヘッダー行（必須）
    - 項目一覧はバージョンごとに固定です。
  - 3行目以降：データ行
    - インポートにおいては 1000行までとします。
    - カンマ（,）を含む文字列データはダブルクォーテーション（"）で囲います。

## データ項目

### Ver1.0

| No. | 項目名 | 必須 | 上限 | 制限 | 備考 |
| --: | -- | :--: | --: | -- | -- |
| 1 | アカウントID | | - | - | ・ ユーザーインポート時は指定不要 |
| 2 | ログイン名 | ◯ | 128文字 | 半角英数字 または 下記の記号のみ<br/>・ハイフン（`-`）<br/>・ピリオド（`.`）<br/>・アンダーバー（`_`）<br/>・アットマーク（`@`） | ・ 組織内で重複不可 <br/> ・ 大文字・小文字の違いは無視する |
| 3 | メールアドレス | ◯ | 128文字 | RFC準拠アドレスを前提とし、半角英数字 または 下記の文字のみ <br/> ・ハイフン（`-`）<br/>・ピリオド（`.`）<br/>・アンダーバー（`_`）<br/>・プラス（`+`）<br/>・パーセント（`%`） | ・ 英字は小文字に変換する <br/> ・ 特殊な形式のメールアドレス（RFC違反アドレス）で使用不可のパターン <br/>　（例1）＠直前ドット：<abcd.@sample.jp> <br/>　（例2）先頭ドット：<.abcd@sample.jp> <br/>　（例3）ドット等の連続記号：<ab..cd@sample.jp> |
| 4 | 表示名 | ◯ | 160文字 | 下記の記号を禁止<br/>・コロン（:）<br/>・ダブルクォーテーション（"） | |
| 5 | 姓 | ◯ | 20文字 |  下記の記号を禁止<br/>・コロン（:）<br/>・ダブルクォーテーション（"） | |
| 6 | 名 | | 20文字 |  下記の記号を禁止<br/>・コロン（:）<br/>・ダブルクォーテーション（"） | |
| 7 | 姓カナ | ◯ | 20文字 |  下記の記号を禁止<br/>・コロン（:）<br/>・ダブルクォーテーション（"） | |
| 8 | 名カナ | | 20文字 |  下記の記号を禁止<br/>・コロン（:）<br/>・ダブルクォーテーション（"） | |

```csv
Ver1.0
アカウントID,ログイン名,メールアドレス,表示名,姓,名,姓カナ,名カナ
,ichiro,ichiro@iidabashi-company.jp,総務部_飯田橋一郎,飯田橋,一郎,イイダバシ,イチロウ
```
