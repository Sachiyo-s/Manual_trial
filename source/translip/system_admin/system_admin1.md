# 取引明細配信のための設定を行う<!-- omit in toc -->

PCA Hub 取引明細を利用するための設定方法です。<br/>
PCA 商魂・商管シリーズ にてあらかじめ連動設定を行う必要があります。<br/>
<br/>
PCA 商魂・商管シリーズの「随時」-「PCA Hub連動設定」から連動設定を行います。<br/>
![1](../assets/system_admin1.image1.png)

## 制限

- PCA Hub との連動設定は PCA Hub 取引明細のシステム管理者のみが行うことができます。
- 配信先の登録で行う取引先同期は、PCA Hub 取引明細のシステム管理者に加え、テナント管理者の権限が必要です。（[テナント管理者を追加する](../tenant_admin/tenant_admin4.md)）

## サービスの URL を設定する

1. 「有効サービス」のチェックをONにして「URL」欄にご契約されている PCA Hub のURLを入力します。<br/>
さらに [移動] ボタンを実行します。<br/>
![2](../assets/system_admin1.image2.png)<br/>

2. 「ブラウザからログイン」を実行します。<br/>
![3](../assets/system_admin1.image3.png)<br/>

3. 表示されたブラウザページにて PCA Hub へのログインを行います。<br/>
![4](../assets/system_admin1.image4.png)<br/>

4. PCA Hub の認可画面にて、要求内容を確認して「同意する」を実行します。<br/>
![5](../assets/system_admin1.image5.png)<br/>

## 明細の公開期間を設定する

PCA Hub 取引明細上に明細を保存する期間を 1～10年 の間で設定します。<br/>
ファイルの公開期間は、アップロード時に指定した公開開始日時 ～ 公開期間の年数となります。<br/>
また、ファイル公開期間の変更は、設定の変更後にアップロードしたファイルにのみ適用されます。<br/>
既にアップロード済みのファイルに対しては適用されません。<br/>
![6](../assets/system_admin1.image6.png)<br/>

## 明細の公開開始日の初期値を設定する

取引明細配信設定ダイアログの公開開始日に設定される初期値を指定します。<br/>
日時指定を指定した場合、アップロードを行った日から指定した日数後の、指定時間に公開されます。<br/>
即時公開を指定した場合、アップロードと同時に公開されます。<br/>
![7](../assets/system_admin1.image7.png)<br/>

## リマインドメールを設定する

公開開始日から指定した日数が経過しても、明細がダウンロードされていない場合、配信先ユーザーに対してリマインドメールを送ることができます。<br/>
通知ユーザーに設定された領域ユーザーが、通知を受け取り、リマインドメールを送信します。<br/>
![8](../assets/system_admin1.image8.png)<br/>

## 案内事項を作成する

案内事項の文書に、配信先ユーザーに対して提示する免責事項などを設定します。<br/>
お客様の運用に適した内容に更新してください。<br/>
設定可能な文字数は下記のとおりです。<br/>

|項目|上限文字数|
|-|-|
|案内事項|2048文字|

## 配信先の登録を行う

明細を配信する配信先を登録します。<br/>
「前準備」-「取引先」-「配信先の登録」から登録することができます。<br/>
個別または一括で登録することができます。<br/>
取引先同期は、テナント管理者の権限が必要です。（[テナント管理者を追加する](../tenant_admin/tenant_admin4.md)）<br/>
  - [単体の配信先の登録を行う](#単体の配信先の登録を行う)
  - [一括で配信先の登録を行う](#一括で配信先の登録を行う)
  - [利用通知送信を行う](#利用通知送信を行う)
  - [取引先同期を行う](#取引先同期を行う)

### 単体の配信先の登録を行う

1. 「新規」ボタンを押します。<br/>
![11](../assets/system_admin1.image11.png)<br/>

2. 各項目を入力し、登録します。<br/>
- 「取引先名」は、クライアントサイトで配信先の社名として表示されます。<br/>
- 「配信先ユーザー」の「ログイン名」は、一意な値を設定してください。他のユーザーと同じログイン名を指定することはできません。<br/>
- 「配信先ユーザー」の「表示名」は、クライアントサイトで担当者名として表示されます。<br/>
![12](../assets/system_admin1.image12.png)<br/>
![22](../assets/system_admin1.image22.png)<br/>

### ポイント<!-- omit in toc -->

- 配信先ユーザーのメールアドレスは、複数の配信先に登録することができます。<br/>
複数の取引先に登録されている場合、ログイン名の背景色が変わります。<br/>
![23](../assets/system_admin1.image23.png)<br/>
- 「請求書配信」が「有効」になっている配信先に対して、請求書配信が行われます。<br/>
他の明細も同様です。<br/>

### 一括で配信先の登録を行う

1. 「一括登録」ボタンを押します。<br/>
![13](../assets/system_admin1.image13.png)<br/>

2. 表示条件を設定します。<br/>
![14](../assets/system_admin1.image14.png)<br/>

3. 各項目を入力し、登録します。<br/>
「作成」にチェックを入れた得意先/仕入先が、配信先として作成されます。<br/>
![15](../assets/system_admin1.image15.png)<br/>

### 利用通知送信を行う

任意で、配信先ユーザーへ「PCA Hub 取引明細」を利用して文書を送付することを、事前に案内するメールを送信することができます。<br/>
他の方法で既に「PCA Hub 取引明細」を利用して文書を送付することを配信先へご案内済みの場合は、送信する必要はありません。<br/>

1. 「前準備」-「メール送信文の登録」から、メール送信文を登録します。<br/>
本文を初期値のまま使用する場合は、メールアドレスや自社担当者名などを適切な値に修正して登録します。<br/>
詳細は、PCA 商魂・商管シリーズのマニュアルをご確認ください。<br/>
![17](../assets/system_admin1.image17.png)<br/>

2. 「前準備」-「取引先」-「配信先の登録」を起動します。<br/>

3. 個別画面、一覧画面から送信したい取引先を選択し「利用通知送信」ボタンを押します。<br/>
4. 利用通知を送信するメール送信文を選択し、「次へ」ボタンを押します。<br/>
![18](../assets/system_admin1.image18.png)<br/>

5. メール設定を行います。<br/>
お客様がお使いのメールを使用して送信を行います。<br/>
設定方法はシステム管理者にお問い合わせください。<br/>
![19](../assets/system_admin1.image19.png)<br/>
![20](../assets/system_admin1.image20.png)<br/>

6. 送信したい配信先ユーザーを選択し、送信ボタンを押すと、利用通知メールが送信されます。<br/>
![21](../assets/system_admin1.image21.png)<br/>

### 取引先同期を行う

新規に登録した配信先データや、配信先の登録内容に追加や修正が入った場合に、PCA Hub 取引明細と同期する必要があります。<br/>
この操作はテナント管理者のみが行えます。<br/>
新しく追加した配信先ユーザーは、同期されるとPCA Hubにアカウントが登録されます。<br/>
**アカウントが登録されると、配信先ユーザーに「パスワード設定リクエスト」「ライセンス付与のお知らせ」のメールが送信されます。**<br/>

1. 個別画面、一覧画面で「Hub取引先同期」ボタンを押します。<br/>

2. 同期を行います。<br/>
アカウントが新規登録された場合には配信先ユーザーに「パスワード設定リクエスト」「ライセンス付与のお知らせ」のメールが送信されます。<br/>
![16](../assets/system_admin1.image16.png)<br/>

3. 同期完了後、必ずデータ領域のバックアップを行ってください。<br/>
「ファイル」-「データ領域のバックアップ」または「データ領域の一括バックアップ」から実行します。<br/>
