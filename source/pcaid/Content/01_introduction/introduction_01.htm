はじめに（PCA ID とは）

当 Web マニュアルは、『PCA ID』 の操作について説明しています。
ご自身の役割や、目的とする操作に応じて、必要な部分をご覧ください。

『PCA ID』 とは？

  • PCA が提供するサービスをより便利に利用して頂けるようにするための “共通ID” です。

  • PCA サービスをより安全に利用して頂けるようになります。

  • お客様へのサポートやご提案、製品・サービスの機能改善に役立てます。

◎ 特徴１

PCA が提供するサービスをより便利に利用して頂けるようにするための “共通ID” です。
PCA Hub サービスを始めとして、PCAクラウドや PCAサブスク、PCA お客様サポートなど、利用できる範囲を順次広げていきます。

  • お客様ごとに1つのアカウントを用意し、PCA が提供する各種サービスをすべて利用可能にしていきます。

  • PCA やそのグループ会社、および提携先会社が提供する、複数の製品やサポートについてのサー ビスを対象とします。

  • 複数の製品やサービス同士がスマートに連携し、お客様がより便利に利用できるようにしていきます。

◎ 特徴２

PCA ID によって、PCA サービスをより安全に利用して頂けるようになります。
PCA サービスのご利用アカウントについて、お客様の管理者に代わって、PCA が安全性を管理・維持することで、乗っ取りやなりすまし等の被害から、お客様を守ります。

  • お客様にとって安全かつ便利な先進認証を採用し、サポートしていきます。

  • 世界標準のセキュリティ基準に基づいて PCA がポリシーを決定し、最新の情報にあわせて更新していきます。

◎ 特徴３

PCA ID によって、お客様のアカウント状態や、製品・サービスの利用状況を把握・分析し、お客様へのサポートやご提案、製品・サービスの機能改善に役立てます。

  • お客様のアカウント状態および利用状況データを以下の目的で利用します。

    1. お客様による製品やサポート利用についてのお問い合わせをサポートするため

    2. お客様に提供している製品・サービスの改善のため

    3. お客様に合わせて最適なサービスのご提案をおこなうため

    4. お客様への最適なマーケティング施策を検討するため

    5. お客様の同意の上で、製品の活用方法などをヒアリングするため

お客様に新たな製品やサービスをご提案し、ご利用頂くため、お客様の管理者が許可したサービスに限定する形で、会社の業務データ、およびお客様が許可した個人情報（社員データ）を共有できるようにします。

  • お客様の業務課題を解決するため、PCA やそのグループ会社、および提携先会社が提供する製品やサービスをご提案します。

  • お客様の個人情報（社員データ）に対しては、プライバシーを最大限に尊重・保護し、お客様自身でいつでも共有を許可または拒否できるようにします。

PCA ID に関する定義・用語

■ PCA サービス、または単に「サービス」

  • PCA が提供する、PCA ID を利用可能なサービス群です。

  • 『PCAクラウド』 『PCA Hub』 等の製品・サービスだけでなく、『お客様サポート』 等のサポートサービスも含めて、お客様に対してワンストップサービスを提供することを想定しています。

■ PCA ユーザー、または単に「ユーザー」

  • PCA サービスの利用者です。

    • 主に顧客企業の社員や業務担当者が対象となります。

    • 士業の先生方や取引先の担当者が対象となることも想定しています。

■ PCA アカウント、または単に「アカウント」

  • PCA ユーザーを識別するための属性群です。

    • ログイン名やメールアドレス、名前、パスワード等の認証情報を含みます。

  • 製品・サービスごとのアカウント作成は不要です。

    • 一人一つのアカウントを使います。

    • パスワード等の認証情報は一元管理できます。

      • 異なる PCA Hub テナントや PCAクラウド 区画であっても、同じパスキーやパスワードを使います。

    • アカウントは共有できません。

■ 組織

  • ユーザーが所属するグループで、PCA サービスの種類に応じて最小グループを構成します。

    • 組織（最小グループ）の例１：PCA Hub テナント 「tenant1」

    • 組織（最小グループ）の例２：PCAクラウド 契約ID 「12345678」

    • 最小グループは「契約」と紐づくことを想定しています。

  • 組織を結合することで、複数の PCA サービスを含む、より大きなグループを構成できます。

    • 複数の PCA サービスに渡ってユーザー管理を統合します。

  • 顧客企業の情報と紐づきます。

    • ユーザーに紐づく顧客企業を特定できます。

    • ユーザーは複数の契約により、複数の顧客企業に紐づくことがあります。

■ 組織管理者

  • 組織ごとの管理者で、組織および組織に所属するユーザーを管理します。

    • 組織に対する管理者権限を持ちます。

  • 導入時は、個々の PCA サービスの管理者が対象者となります。
